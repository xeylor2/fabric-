## Overview

FEBRIC is a **professional AI textile design operating system** — not an AI
image generator. Every garment element (motifs, borders, pallus, sleeves,
necklines, grounds) is an independently editable, typed node, and AI acts only
as a scoped, user-approved assistant that never holds the pen.

`v2.0.0` is the **first official release** and establishes the permanent
Architecture V2 baseline as the single public source of truth. It supersedes
the unreleased V1 prototype in full. This is a **foundation release**: the
engine substrate is built and frozen; end-user editing surfaces arrive in
subsequent milestones.

## Architecture Summary

- **Clean Architecture** per module (`presentation → domain ← data`,
  inward-only) and the global rule **`app → features → core → nothing`** —
  features never import features. Enforced in CI by
  `tooling/dependency_lint.dart`.
- **The Design Tree is the spine**: the document *is* a semantic tree
  (`Project → Artwork → Garment → GarmentPart → Zone → Content`), and every
  system is defined by its relationship to that tree.
- **One mutation path**: the command bus. Human drag, AI replace, and natural
  language all become serializable `DocumentCommand`s.
- **Constitutional invariants I1–I6** (one bus, staging before apply, scope as
  authority, absolute locks, mandatory provenance, distinct whole-design
  regen) are testable, permanent, and CI-enforced.
- **Pure-Dart core tier** (isolate/backend-safe) with a dependency-free
  foundation tier (`core_common`, `core_geometry`, `core_textile`,
  `core_lock`, `core_plugin`).

## Implemented Engines

- **Document Engine** (`core_document`) — `FebricDocument` single source of
  truth; frozen pipeline **Lock → Validation → History → Document**;
  command-based undo/redo; `.febric` versioned format with migrations.
- **Design Tree Engine** (`core_document`) — 8 tree commands on the one bus;
  `NodeCapabilityMatrix` (12 capabilities); `SemanticPath` addressing.
- **Asset Engine** (`core_asset`) — single asset identity shared with the
  document manifest; content-addressed immutable store; append-only version
  lineage; usage graph with reference counting.
- **Frozen foundations** — Lock, Layer, Mask, Selection, Plugin SDK,
  Performance, Universal Tool Contract (14 members), Universal Canvas Contract,
  Prompt Compiler (10 stages), Universal AI Pipeline (13 stages),
  `core_geometry`, `core_interaction`, `core_textile`.
- **Theme Engine** (`core_design_system`) and **Adaptive Workspace Shell**
  (`core_workspace`).

## Known Limitations

Foundation release — **no rendered canvas, live editing, AI execution, or
export yet**. Feature modules beyond shell/theme/settings are placeholders.
`core_ai` is contracts-only; `backend/` is a placeholder. Windows desktop
build requires Visual Studio locally (CI covers it).
