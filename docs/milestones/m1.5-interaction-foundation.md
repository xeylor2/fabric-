# M1.5 — Universal Workspace Interaction Foundation (2026-07-07)

New pure-Dart foundation package **`core_interaction`** (ADR-0003): the one
interaction system every editing surface inherits. No canvas, no rendering,
no AI, no tool logic — immutable models, contracts and pure behaviour only.

## Delivered

| Area | Types |
|---|---|
| Geometry | `Point2D`, `Size2D`, `Rect2D` (freezed, JSON; own primitives — pure Dart cannot use `dart:ui`) |
| Viewport | `ViewportState` — zoom, offset, rotation, canvasSize, screenSize, devicePixelRatio, derived `visibleRect` (rotation-aware); exact canvas↔screen transforms |
| Zoom rules | `ZoomCommand` (frozen 9): default, fit screen, actual size, to selection, to object, to cursor, in, out, reset — implemented by pure `ViewportOps`, always clamped via `ZoomConstraints` |
| Camera | `ViewportAnimation` (smooth zoom: **geometric** interpolation, eased curves), `InertiaSpec` (exponential-decay fling), `NavigationState` (bounded view history) |
| Gestures | `GestureKind` (frozen 11: tap, double tap, long press, drag, pan, pinch zoom, two-finger pan, rotate, mouse-wheel zoom, trackpad pinch, keyboard shortcut), sealed `InteractionEvent` union, `GesturePhase`, `InputDevice` (touch/mouse/trackpad/stylus), `KeyModifiers`, `InteractionShortcut` |
| Policy | `ViewportInteractionMapper` — one universal event→viewport mapping (anchored pinch/wheel zoom, smart double-tap, Figma-style wheel config) + `InteractionConfig` |
| Selection | `SelectionState` + `SelectionMode` with the universal modifier policy (primary toggles, shift adds, bare replaces) |

## Platform rules, made enforceable

"Identical behaviour everywhere" is a **test suite**, not a promise:
touch pinch ≡ trackpad pinch ≡ Ctrl/Cmd+wheel (same zoom, same anchor);
Ctrl ≡ Cmd; two-finger pan ≡ pointer pan; content gestures (tap/drag/long
press) never move the viewport. Devices are data, never behaviour branches.
