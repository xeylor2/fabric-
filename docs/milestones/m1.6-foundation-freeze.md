# M1.6 — FEBRIC Foundation Freeze (2026-07-07)

Architecture freeze completed **before** the Document Engine exists. Ten
engines frozen as pure architecture — immutable freezed models, typed
interfaces, ADRs — with **zero implementation, zero UI, zero rendering,
zero business logic**. Master document:
`docs/architecture/foundation-freeze.md` (diagrams, dependency rules, API
surface, migration, acceptance, testing strategy).

## Structural guarantees (typed, not conventional)

- `ProviderAdapter.submit(CompiledPrompt)` — **no provider can receive raw
  conversation**; the type system enforces the mandate.
- `AiPipelineRun.advanceTo` throws on any out-of-order stage — document
  update is unreachable before validation; approval unreachable before
  review.
- `LockSet` implication closure makes I4 checkable before compute.
- The 14-member tool contract is pinned by an end-to-end implementability
  test; plugins reach engines only through JSON command dispatch.
